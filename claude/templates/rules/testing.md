# Testing Rules

- Verify your own work — run tests, lint, type checks after changes. Don't assume it works.
- Run the actual test/lint command. IDE diagnostics alone are not sufficient.
- When the user says "test it", run the script — don't substitute log checks or code inspection.
- After 3 failed attempts at fixing the same issue, verify every subsequent change works before declaring it done.
- Never suggest stopping testing early. The user decides when there is enough proof.
- Never suggest committing until the change has been tested and verified.
