# Git Rules

- Commits: Conventional Commits (`feat|fix|refactor|docs|test|chore`).
- Write a concise message that focuses on the "why" rather than the "what".
- Never commit without running lint/tests first.
- Never suggest committing or pushing until the user asks. Testing comes first, always.
- The user handles deployments — "I will deploy this" means wait.
- Don't amend unless asked. Create new commits.
- Don't force-push, reset --hard, or destructive ops without explicit ask.
